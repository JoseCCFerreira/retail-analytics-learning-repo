2) Git e GitHub — Como usar aqui

Fluxo base para trabalhar com segurança

Passo 1: Actualizar branch principal
git checkout main
git pull origin main
Passo 2: Criar branch de trabalho
git checkout -b feat/ml-central-hub
Passo 3: Implementar mudanças e testar
Passo 4: Adicionar e commitar
git add .
git commit -m "feat: add centralized project hub and ML workflow"
Passo 5: Publicar branch
git push -u origin feat/ml-central-hub
Passo 6: Abrir Pull Request no GitHub

Comandos úteis no dia-a-dia

git status
git branch
git log --oneline --decorate --graph -n 20
git diff
git diff --staged
git restore --staged <ficheiro>    # tirar do stage
git checkout -- <ficheiro>          # descartar alterações locais

Para reverter um commit já publicado de forma segura:

git revert <hash_do_commit>

Boas práticas de commit

  • Um commit = uma intenção clara
  • Mensagens no formato: feat:, fix:, docs:, test:, refactor:
  • Executar testes antes de commitar
  • Não misturar mudanças de docs + código crítico no mesmo commit, quando possível

Modelo de Pull Request (GitHub)

Título:
feat: centralize project guide and validation matrix

Descrição:
- add centralized index with sections for Data/Modeling/ML/Viz/Tests
- add Git and GitHub workflow instructions
- include validation commands for each layer

Checklist:
[ ] Data pipeline executed
[ ] dbt run and dbt test OK
[ ] ML pipeline executed and outputs generated
[ ] Streamlit app opened successfully
[ ] Docs links verified
Se estiveres a aprender, faz commits pequenos e frequentes. É mais fácil perceber erros e voltar atrás sem dor.

4) Caso Realista de Teste — Black Friday 2025

Contexto de negócio: retalho alimentar com campanha forte de Black Friday, aumento de tráfego, descontos agressivos em categorias específicas e mudança no mix de pagamentos.

O que pretendemos seguir

  • Receita: pico no dia 2025-11-28 vs baseline de Novembro
  • Desconto: aumento do discount ratio no período 2025-11-28..2025-12-01
  • Pagamentos: subida de MBWay após 2025-11-27
  • Mix de categorias: snacks e cleaning com maior peso na campanha

O que esperamos ver

  • Black Friday com uplift forte de receita diária
  • Margem pressionada por maior desconto médio
  • Mudança de comportamento no checkout (menos cash, mais mobile)
  • Base ideal para validar modelos de classificação e segmentação

Ficheiros do caso

  • python/generate_test_case_black_friday.py — gera dataset e DuckDB do caso
  • python/analyze_test_case_black_friday.py — compara baseline vs evento e imprime sinais
  • streamlit/case_black_friday_app.py — dashboard visual baseline vs campanha
  • data/test_cases/black_friday_2025/expected_signals_case.csv — sinais esperados

Como correr

npm run case:generate
npm run case:analyze
npm run case:dashboard
Este caso é semi-sintético mas com dinâmica realista de negócio. Serve para experimentar hipóteses e validar se os pipelines e modelos conseguem capturar sinais esperados de campanha.

6) Data Modeling — dbt

Objectivo

Aplicar modelação com camadas staging e marts usando dbt, garantindo qualidade com testes em schema.yml.

  • staging: materialized as view
  • marts: materialized as table

Ficheiros-chave

  • dbt_retail_analytics/dbt_project.yml
  • dbt_retail_analytics/models/schema.yml
  • dbt_retail_analytics/models/marts/fct_sales.sql

Comandos dbt

cd dbt_retail_analytics

# 1) Garantir profile configurado a partir de profiles.example.yml
# 2) Executar modelos
dbt run

# 3) Executar testes de qualidade
dbt test

# 4) Ver documentação (opcional)
dbt docs generate
dbt docs serve

Testes de Data Modeling existentes

  • not_null + unique para chaves como transaction_id e sales_item_id
  • relationships para integridade referencial entre fact e staging
Se dbt test falhar, valida upstream data primeiro (CSV/duckdb) antes de mexer em SQL dos modelos.

7) Machine Learning — Supervised + Unsupervised

Componentes

  • Script principal: python/ml_retail.py
  • Guia detalhado: docs/machine_learning_guide.html
  • Output: data/ml_outputs/

Modelos incluídos

  • Regressão: Linear Regression, Decision Tree, Random Forest, Gradient Boosting
  • Classificação: Logistic Regression, Decision Tree, Random Forest
  • Clustering: K-Means + DBSCAN
  • Redução dimensional: PCA
  • Validação: Cross-validation 5-fold

Comando para executar ML

python python/ml_retail.py

8) Data Visualization — Streamlit + Plotly

O que existe no dashboard

  • KPIs: total transações, receita total, valor médio por linha
  • Série temporal: receita por mês
  • Top produtos por receita

Fonte: DuckDB via streamlit/app.py

Executar visualização

streamlit run streamlit/app.py

Depois abrir o URL mostrado no terminal (normalmente localhost:8501).

Validações de visualização

# smoke test rápido do streamlit script
python -m py_compile streamlit/app.py

# confirmar db disponível para a app
python -c "from pathlib import Path; print(Path('data/retail_analytics.duckdb').exists())"
Se a app mostrar aviso de base em falta, executa primeiro python python/run_pipeline.py.
