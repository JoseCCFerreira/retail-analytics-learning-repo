1. Porque adicionar PyTorch e TensorFlow

O projeto atual já te dá uma base ótima com scikit-learn, SQL, dbt e dashboards. O próximo passo natural para aprender mais fundo é entrares em Deep Learning: redes neuronais, embeddings, séries temporais mais ricas, recomendação e modelos mais flexíveis.

2. Metodologia de Aprendizagem

Em vez de saltar diretamente para código complexo, usa esta ordem:

  1. Definir o problema de negócio.
  2. Traduzir o problema para um tipo de aprendizagem.
  3. Perceber a teoria mínima do método antes de programar.
  4. Começar com baseline simples.
  5. Só depois testar rede neuronal.
  6. Comparar sempre com métricas e interpretação.
  7. Guardar experiência, resultados e lições aprendidas.
A metodologia certa é: teoria pequena, prática curta, comparação honesta, iteração controlada.

4. Base Teórica que deves dominar

4.1 Tipos de aprendizagem

  • Supervisionada: tens exemplos com rótulos.
  • Não supervisionada: procuras padrões sem rótulos.
  • Semi-supervisionada: misturas poucos rótulos com muitos dados sem rótulo.
  • Self-supervised: o próprio dado gera o objetivo de treino.

4.2 Conceitos centrais

  • Feature: coluna usada pelo modelo.
  • Target: o que queres prever.
  • Loss function: mede o erro do modelo.
  • Gradient descent: método de otimização.
  • Overfitting: o modelo memoriza demais.
  • Regularização: técnicas para melhorar generalização.
  • Batch, epoch, learning rate: pilares do treino em redes neuronais.

4.3 O que muda no Deep Learning

No ML clássico, tu escolhes muitas features manualmente. No Deep Learning, a rede aprende representações internas. Isso aumenta flexibilidade, mas também aumenta custo, necessidade de dados e risco de complexidade desnecessária.

5. Aprender com PyTorch

PyTorch é excelente para perceberes o treino quase “por dentro”.

Instalação opcional

pip install -r requirements_deep_learning.txt

Estrutura mental de um projeto PyTorch

  1. Carregar dados.
  2. Transformar em tensores.
  3. Criar Dataset e DataLoader.
  4. Definir a rede.
  5. Escolher loss e optimizer.
  6. Treinar por épocas.
  7. Avaliar e guardar métricas.

Exemplo mínimo

import torch
import torch.nn as nn
import torch.optim as optim

model = nn.Sequential(
    nn.Linear(10, 32),
    nn.ReLU(),
    nn.Linear(32, 1)
)

criterion = nn.MSELoss()
optimizer = optim.Adam(model.parameters(), lr=0.001)

O que estás a aprender aqui

  • Camadas lineares.
  • Funções de ativação.
  • Propagação para a frente.
  • Backpropagation.
  • Atualização de pesos.

6. Aprender com TensorFlow / Keras

TensorFlow com Keras é muito bom para aprender fluxos de treino organizados e construir modelos mais rapidamente.

Exemplo mínimo

import tensorflow as tf
from tensorflow import keras

model = keras.Sequential([
    keras.layers.Dense(32, activation="relu", input_shape=(10,)),
    keras.layers.Dense(1)
])

model.compile(
    optimizer="adam",
    loss="mse",
    metrics=["mae"]
)

Quando TensorFlow é útil para aprender

  • Quando queres treinar com callbacks.
  • Quando queres early stopping facilmente.
  • Quando queres ver uma API de treino mais “alta”.
  • Quando queres usar Keras para prototipagem rápida.
