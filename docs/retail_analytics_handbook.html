Se o comando gh não existir, instala-o antes de tentares criar repositórios via terminal.

Sem Homebrew, instalação local no utilizador:

cd ~/.local/gh

5. OLTP - SQLite

OLTP significa Online Transaction Processing. É a base do dia a dia do negócio, onde cada registo representa uma operação concreta.

Objetivo

  • guardar clientes, produtos, lojas, transações e itens vendidos;
  • preservar integridade com chaves primárias e estrangeiras;
  • simular uma base de aplicação operacional.

Comandos

python python/generate_data.py
python python/setup_sqlite.py

Exemplo de esquema

CREATE TABLE transactions (
    transaction_id INTEGER PRIMARY KEY,
    transaction_date TEXT NOT NULL,
    customer_id INTEGER NOT NULL,
    store_id INTEGER NOT NULL,
    payment_method TEXT NOT NULL,
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id),
    FOREIGN KEY (store_id) REFERENCES stores(store_id)
);

6. OLAP - DuckDB

OLAP significa Online Analytical Processing. Serve para análise, agregações e perguntas de negócio sobre muitos registos ao mesmo tempo.

Objetivo

  • criar uma tabela de factos fct_sales;
  • juntar transações com linhas de venda;
  • preparar dados para dashboards, dbt e ML.

Comandos

python python/setup_duckdb.py

Exemplo de fact table

CREATE OR REPLACE TABLE fct_sales AS
SELECT
    si.sales_item_id,
    t.transaction_id,
    CAST(t.transaction_date AS DATE) AS transaction_date,
    t.customer_id,
    t.store_id,
    t.payment_method,
    si.product_id,
    si.quantity,
    si.unit_price,
    si.discount_pct,
    si.gross_amount,
    si.discount_amount,
    si.net_amount
FROM sales_items si
INNER JOIN transactions t
    ON t.transaction_id = si.transaction_id;

7. dbt - Governança de Dados

dbt organiza as transformações em camadas, mantém SQL versionado e permite testes de qualidade.

Camadas

  • staging: limpeza e normalização;
  • marts: tabelas finais para análise e consumo;
  • schema.yml: regras e testes.

Executar

cd dbt_retail_analytics
cp profiles.example.yml profiles.yml
export DBT_PROFILES_DIR=.
dbt run
dbt test

Exemplo de teste

- name: transaction_id
  tests:
    - not_null
    - relationships:
        arguments:
          to: ref('stg_transactions')
          field: transaction_id

9. Machine Learning com scikit-learn

Problemas trabalhados

Problema Tipo Modelos
Prever valor da venda Regressão Linear Regression, Decision Tree, Random Forest, Gradient Boosting
Prever segmento do cliente Classificação Logistic Regression, Decision Tree, Random Forest
Agrupar clientes Clustering K-Means, DBSCAN, PCA

Executar

python python/ml_retail.py

Saídas

  • data/ml_outputs/model_metrics.json
  • data/ml_outputs/regression_feature_importance.csv
  • data/ml_outputs/cluster_assignments.csv
  • data/ml_outputs/pca_projections.csv

Base teórica mínima

  • MAE: erro médio absoluto;
  • RMSE: penaliza mais os erros grandes;
  • R2: quanto da variabilidade é explicada;
  • Accuracy: percentagem de acertos;
  • Silhouette Score: qualidade dos clusters.

10. Deep Learning com PyTorch e TensorFlow

Depois de dominares os baselines com scikit-learn, entra o passo seguinte: redes neuronais sobre os mesmos dados para comparar com honestidade.

Metodologia correta

  1. definir o problema de negócio;
  2. criar um baseline simples em scikit-learn;
  3. escolher a métrica antes do treino;
  4. testar uma MLP pequena;
  5. comparar resultados com o baseline;
  6. só manter Deep Learning se trouxer valor real.

Ferramentas

Ferramenta Uso
PyTorchAprender treino manual, tensores, DataLoader e redes.
TensorFlow / KerasTreino mais declarativo e rápido de prototipar.

11. PL/SQL e Oracle

A pasta sql/oracle_plsql_concepts existe para estudo. SQLite e DuckDB não executam PL/SQL, por isso esta parte é teórica ou para usar num ambiente Oracle real.

Bloco exemplo

SET SERVEROUTPUT ON;

DECLARE
    v_total NUMBER;
BEGIN
    SELECT COUNT(*)
    INTO v_total
    FROM transactions;

    DBMS_OUTPUT.PUT_LINE('Total de transacoes: ' || v_total);
END;
/

Objetivos de estudo

  • procedures;
  • triggers;
  • exceptions;
  • diferença entre OLTP e OLAP em contexto Oracle.
