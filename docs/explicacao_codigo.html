Explicação do Código — visão completa

Este documento explica o papel de cada ficheiro de código criado no projeto e como tudo se conecta no pipeline.

1) Scripts Python

python/generate_data.py

  • Gera dados sintéticos de lojas, produtos, clientes, transações e itens de venda.
  • Escreve CSVs em data/ com seed fixa para reprodutibilidade.

python/setup_sqlite.py

  • Cria base operacional SQLite.
  • Executa scripts SQL de criação de tabelas e índices.
  • Carrega CSVs para tabelas relacionais.

python/setup_duckdb.py

  • Cria base analítica DuckDB.
  • Carrega dados de CSV e cria tabela analítica consolidada (fct_sales).

2) SQL

sql/sqlite/01_create_tables.sql

Define o modelo relacional operacional com PK/FK e integridade básica.

sql/sqlite/02_indexes.sql

Cria índices para filtros e joins frequentes.

sql/sqlite/03_business_queries.sql

Consultas de negócio para receita, ticket médio e ranking de produtos.

sql/duckdb/01_analytical_queries.sql

Consultas analíticas avançadas com CTEs e funções de janela.

sql/oracle_plsql_concepts/01_plsql_study_case.sql

Exemplos de procedure, function, trigger e package no estilo Oracle PL/SQL.

3) dbt

O projeto dbt organiza o pipeline em três níveis:

  • staging: normalização inicial dos campos de origem.
  • marts: fact table e dimensões para consumo analítico.
  • schema tests: validação de integridade (not_null, unique, relationships).

4) Streamlit

streamlit/app.py

  • Lê a base DuckDB e exibe KPIs principais.
  • Mostra evolução de receita por mês e ranking de produtos.
  • Entrega uma interface simples para validação rápida dos dados.

5) Como executar ponta a ponta

pip install -r requirements.txt
python python/generate_data.py
python python/setup_sqlite.py
python python/setup_duckdb.py
streamlit run streamlit/app.py
