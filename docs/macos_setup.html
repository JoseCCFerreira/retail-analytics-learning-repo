Nota macOS: no macOS o comando é quase sempre python3 e pip3 em vez de python e pip. Dentro do ambiente virtual, ambos ficam disponíveis como python e pip.

1. Instalar Python 3.10+

Opção A — Homebrew (recomendado)

Se ainda não tens Homebrew, instala primeiro (ver secção 2). Depois:

brew install python@3.12

Adiciona ao PATH (coloca no final do teu ~/.zshrc ou ~/.bash_profile):

export PATH="$(brew --prefix python@3.12)/bin:$PATH"

Aplica as alterações:

source ~/.zshrc

Confirma:

python3 --version
# Python 3.12.x

Opção B — Instalador oficial

Download em python.org/downloads/macos. Corre o .pkg e segue o instalador. Depois confirma:

python3 --version

Opção C — pyenv (melhor para múltiplas versões)

brew install pyenv
echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.zshrc
echo 'export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.zshrc
echo 'eval "$(pyenv init -)"' >> ~/.zshrc
source ~/.zshrc

pyenv install 3.12.3
pyenv global 3.12.3
python3 --version

2. Instalar Homebrew (opcional mas recomendado)

Homebrew é o gestor de pacotes do macOS. Facilita muito a instalação de Python, Git e outras ferramentas.

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Após instalação, em Macs com chip Apple Silicon (M1/M2/M3/M4) adiciona ao PATH:

echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zshrc
source ~/.zshrc
brew --version
Intel vs Apple Silicon: em Macs Intel, o Homebrew instala em /usr/local. Em Apple Silicon (M1+), instala em /opt/homebrew. O comando acima funciona para ambos.

3. Clonar o repositório

Confirma que tens Git:

git --version
# Se não estiver: xcode-select --install  (instala Xcode CLI Tools com Git incluído)

Clona o repositório:

git clone <URL_DO_REPOSITORIO>
cd retail_analytics_learning_repo
Se já tens a pasta localmente, abre o Terminal, navega até à pasta com cd ou arrasta a pasta para o Terminal para auto-completar o caminho.

4. Criar e ativar ambiente virtual

1

Cria o venv

python3 -m venv .venv

Cria a pasta .venv/ com Python isolado dentro do projeto.

2

Ativa o venv

source .venv/bin/activate

O prompt do Terminal passa a mostrar (.venv) no início. Isso confirma que está ativo.

3

Confirma Python ativo

which python
# deve mostrar: .../retail_case_repo.../`.venv/bin/python`
Para desativar o venv quando acabares: deactivate
Para reativar em sessões futuras: source .venv/bin/activate

6. Executar o pipeline de dados

Tudo de uma vez

python python/run_pipeline.py

Ou passo a passo

1

Gerar CSVs com dados simulados

python python/generate_data.py

Cria data/customers.csv, products.csv, stores.csv, transactions.csv, sales_items.csv.

2

Criar base SQLite

python python/setup_sqlite.py

Resultado: data/retail_app.db

3

Criar base DuckDB

python python/setup_duckdb.py

Resultado: data/retail_analytics.duckdb

Validar

ls -lh data/
# Deves ver retail_app.db e retail_analytics.duckdb

7. Configurar e executar dbt

1

Cria o diretório de perfis dbt

mkdir -p ~/.dbt
2

Cria o ficheiro profiles.yml

cat << 'EOF' > ~/.dbt/profiles.yml
retail_analytics:
  target: dev
  outputs:
    dev:
      type: duckdb
      path: /CAMINHO/ABSOLUTO/data/retail_analytics.duckdb
      threads: 4
EOF

Substitui /CAMINHO/ABSOLUTO pelo caminho real. Para o saber:

pwd
# copia o resultado e substitui acima
3

Executa dbt

cd dbt_retail_analytics
dbt run
dbt test
dbt docs generate && dbt docs serve

8. Executar o Streamlit

streamlit run streamlit/app.py

Abre automaticamente o browser em http://localhost:8501.

Para parar: Ctrl + C no Terminal.

Dica macOS: para abrir o browser manualmente: open http://localhost:8501

9. Servir documentação local

Na raiz do projeto:

python -m http.server 8080

Abre: open http://localhost:8080

Com Node.js (opcional):

npm install
npm run docs:serve

10. Troubleshooting macOS específico

python3: command not found

Instala Python via Homebrew (brew install python@3.12) ou via instalador oficial.

pip: command not found dentro do venv

python -m ensurepip --upgrade

Erro de compilação em dependências

Instala as Xcode Command Line Tools:

xcode-select --install

Apple Silicon (M1/M2/M3/M4): conflitos de arquitetura

Garante que estás a usar Python nativo ARM e não Rosetta:

python3 -c "import platform; print(platform.machine())"
# deve mostrar: arm64

Se mostrar x86_64, estás em modo Rosetta. Reinstala Python via Homebrew nativo ARM.

dbt: command not found

Confirma que o venv está ativo (source .venv/bin/activate) e corre pip install -r requirements.txt.

DuckDB: path does not exist

Usa o caminho absoluto no profiles.yml em vez de caminho relativo.

Port 8080 ou 8501 já em uso

lsof -i :8080   # ver o que está a usar a porta
kill -9 <PID>  # encerrar o processo

Ou usa outra porta: python -m http.server 8090
