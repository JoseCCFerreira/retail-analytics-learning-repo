Guia de Instalação Completo

Este guia prepara o repositório de ponta a ponta: ambiente local, dependências, estrutura recomendada, execução de scripts, dbt, documentação e validação final.

macOS? Guia dedicado com Homebrew, Apple Silicon e troubleshooting completo: Guia macOS →

1) Pré-requisitos

  • Windows 10/11, macOS ou Linux
  • Python 3.10+ instalado e no PATH
  • Git instalado
  • Acesso a terminal (PowerShell, CMD, bash ou zsh)
  • Opcional: Node.js 18+ para usar scripts do package.json

2) Clonar e abrir o repositório

git clone <URL_DO_REPOSITORIO>
cd retail_analytics_learning_repo
Se já tiveres o repositório local, entra apenas na pasta raiz do projeto.

3) Criar e ativar ambiente virtual

Windows (PowerShell)

python -m venv .venv
.\.venv\Scripts\Activate.ps1

macOS / Linux

python3 -m venv .venv
source .venv/bin/activate

macOS — guia detalhado com Apple Silicon, Homebrew e pyenv: docs/macos_setup.html →

4) Instalar dependências Python (requirements)

python -m pip install --upgrade pip
pip install -r requirements.txt

O ficheiro requirements.txt inclui os pacotes necessários para dados, SQL analytics, dbt e visualização.

5) Dependências do package (scripts de docs)

O repositório contém package.json com scripts utilitários para servir a documentação local.

npm install
npm run docs:serve
Alternativa sem Node.js: python -m http.server 8080.

6) Estrutura de código recomendada

.
├── data/
├── python/
│   ├── generate_data.py
│   ├── setup_sqlite.py
│   └── setup_duckdb.py
├── sql/
│   ├── sqlite/
│   ├── duckdb/
│   └── oracle_plsql_concepts/
├── dbt_retail_analytics/
├── streamlit/
├── docs/
├── requirements.txt
└── package.json

Os documentos desta pasta assumem esta convenção de organização.

7) Pipeline técnico (quando os scripts estiverem presentes)

  1. Gerar dados simulados: python python/generate_data.py
  2. Criar BD SQLite: python python/setup_sqlite.py
  3. Criar BD DuckDB: python python/setup_duckdb.py
  4. Executar modelos dbt: dbt seed && dbt run && dbt test
  5. Levantar app Streamlit: streamlit run streamlit/app.py
Se algum diretório ainda não existir, usa este guia como blueprint para implementação faseada.

8) Configuração dbt (DuckDB)

Cria/edita o ficheiro profiles.yml no diretório de perfis dbt:

  • Windows: %USERPROFILE%\.dbt\profiles.yml
  • Linux/macOS: ~/.dbt/profiles.yml
retail_analytics:
  target: dev
  outputs:
    dev:
      type: duckdb
      path: ../data/retail_analytics.duckdb
      threads: 4

9) Checklist de validação final

  • pip install -r requirements.txt termina sem erros
  • index.html abre corretamente no browser
  • Guias em docs/ navegáveis e consistentes
  • Scripts de documentação executáveis por npm run docs:serve ou python -m http.server 8080
  • Se houver código de pipeline, execução ponta a ponta validada
Resultado esperado: ambiente pronto, requisitos definidos, documentação completa e base preparada para desenvolvimento e apresentação.

10) Troubleshooting rápido

Erro ao ativar venv no PowerShell

Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

dbt não encontrado

Confirma ambiente virtual ativo e reinstala com pip install -r requirements.txt.

npm não encontrado

Instala Node.js LTS ou usa alternativa com python -m http.server 8080.

macOS — python3: command not found

brew install python@3.12

macOS — erro de compilação em dependências

xcode-select --install

macOS Apple Silicon — conflito de arquitetura

python3 -c "import platform; print(platform.machine())"
# deve devolver arm64

DuckDB: path error no profiles.yml (macOS)

Usa caminho absoluto. Obtém-no com pwd na raiz do projeto.
