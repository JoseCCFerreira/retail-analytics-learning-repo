1 O que é Machine Learning?

Machine Learning (ML) é a área da Inteligência Artificial que permite a computadores aprender padrões a partir de dados sem serem explicitamente programados com regras fixas.

Analogia simples: Imagine que és novo num supermercado. No primeiro dia perguntas ao gestor o preço de cada produto (regras explícitas). Ao fim de um mês, já sabes estimar o preço de um produto desconhecido pelo tamanho, marca, e categoria. Isso é ML — aprendeste padrões dos dados.

As três grandes categorias de ML

👨‍🏫

Supervisionado

Treinamos com dados que têm resposta certa (label). O modelo aprende a mapear input → output. Exemplo: prever preço de venda, classificar cliente.

🔍

Não-Supervisionado

Os dados não têm labels. O modelo descobre estrutura oculta por conta própria. Exemplo: agrupar clientes por comportamento.

🎮

Por Reforço

O modelo aprende através de tentativa e erro recebendo recompensas/punições. Exemplo: AlphaGo, carros autónomos. (fora do scope deste projecto)

4 Feature Engineering

Feature Engineering é o processo de transformar dados brutos em features (variáveis de entrada) que os algoritmos de ML conseguem processar. É frequentemente o passo mais importante — melhores features > melhores modelos.

Regra de ouro: Os modelos de ML só trabalham com números. Strings como "beverage" ou "budget" têm de ser convertidas para inteiros.

# 1. Datas → componentes numéricas
df["day_of_week"] = df["transaction_date"].dt.dayofweek   # 0=Segunda, 6=Domingo
df["month"]       = df["transaction_date"].dt.month       # 1-12
df["is_weekend"]  = (df["day_of_week"] >= 5).astype(int)  # 0 ou 1

# 2. Strings → inteiros (LabelEncoder)

df["category_encoded"] = le.fit_transform(df["category"])
# "bakery"→0, "beverage"→1, "cleaning"→2, "dairy"→3, "frozen"→4, "snacks"→5

# 3. Normalização para modelos lineares
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X_train)   # média=0, desvio_padrão=1
⚠️ Cuidado: Data Leakage
Nunca uses scaler.fit() nos dados de teste! O scaler deve ser ajustado apenas no treino e depois aplicado ao teste. Se fizeres fit no conjunto completo, o modelo "vê" o futuro e as métricas ficam infladas artificialmente.

5 Aprendizagem Supervisionada — Conceito

Na aprendizagem supervisionada temos um conjunto de dados onde cada exemplo tem uma resposta certa (chamada de label ou target). O modelo aprende uma função que mapeia features (X) → target (y).

TREINO: ┌────────────────────────────────────┬──────────────┐ │ X (features) │ y (target) │ ├────────────────────────────────────┼──────────────┤ │ qty=2, price=4.5, cat=dairy, ... │ net=8.55 │ │ qty=1, price=22.0, cat=frozen, ... │ net=18.70 │ │ qty=3, price=1.2, cat=bakery, ... │ net=3.42 │ │ ... │ ... │ └────────────────────────────────────┴──────────────┘ │ model.fit(X_train, y_train) ▼ MODELO TREINADO │ model.predict(X_new) ▼ ┌───────────────────────────────────────────────────┐ │ X_new: qty=2, price=5.0, cat=snacks, ... │ │ ŷ (previsão) = 9.25 € │ └───────────────────────────────────────────────────┘

🔢 Regressão

O target é um número contínuo.

  • Prever o valor de uma venda (€)
  • Prever temperatura amanhã
  • Prever preço de casa

Métricas: MAE, RMSE, R²

🏷️ Classificação

O target é uma categoria/classe.

  • Prever segmento do cliente (budget/premium/…)
  • Spam vs não-spam
  • Doença vs saudável

Métricas: Accuracy, F1, Precision, Recall

6 Regressão — Prever net_amount

Queremos prever quanto vale cada linha de venda (net_amount) dado o produto, quantidade, desconto, data, cliente e loja. Testamos 4 modelos com complexidade crescente.

Modelo 1: Linear Regression

Teoria: Ajusta uma hiperplano linear aos dados. Equação: ŷ = β₀ + β₁×qty + β₂×price + β₃×discount + ... Cada β (coeficiente) indica o impacto de cada feature na previsão.
  • Vantagens: Extremamente rápido, interpretável, bom como baseline
  • Desvantagens: Assume relação linear — falha em padrões complexos
  • Quando usar: Sempre como primeiro modelo de comparação
from sklearn.linear_model import LinearRegression

model = LinearRegression()
model.fit(X_train_scaled, y_train)    # aprende coeficientes
predictions = model.predict(X_test_scaled)

# Coeficientes: quanto cada feature influencia net_amount
for feat, coef in zip(feature_names, model.coef_):
    print(f"  {feat:25s}: {coef:+.4f}")

Modelo 2: Decision Tree Regressor

Teoria: Divide os dados em regiões através de if/else recursivos. Em cada nó pergunta: "feature X > threshold?" e divide em 2 ramos. A previsão final é a média do y na folha.
unit_price > 10? ├── SIM (acima de €10) │ discount_pct > 0? │ ├── SIM → média_net = 18.50 € │ └── NÃO → média_net = 22.10 € └── NÃO (abaixo de €10) quantity > 2? ├── SIM → média_net = 5.80 € └── NÃO → média_net = 3.20 €
  • Vantagens: Visualmente interpretável, captura não-linearidades
  • Desvantagens: Tende a overfit (memoriza dados de treino), instável
  • max_depth=6: Limitamos profundidade para controlar overfit

Modelo 3: Random Forest Regressor

Teoria: Constrói N=100 árvores de decisão diferentes (cada uma treinada em amostra aleatória dos dados e features). A previsão final é a média de todas as árvores. Isso reduz drásticamente o overfit — chama-se ensemble learning e bagging.
Dados de treino │ bootstrap sampling (cada árvore usa ~63% dos dados aleatoriamente) │ ┌────┴────┐ │ Árvore 1│ → ŷ₁ = 12.40 │ Árvore 2│ → ŷ₂ = 11.80 │ Árvore 3│ → ŷ₃ = 12.95 │ ... │ │Árvore 100│ → ŷ₁₀₀ = 12.20 └─────────┘ │ Média → ŷ_final = 12.47 € (muito mais estável que 1 árvore só)
  • Feature Importance: O RF calcula automaticamente quais features mais contribuem → guardamos em regression_feature_importance.csv
  • n_jobs=-1: usa todos os núcleos da CPU para paralelizar treino

Modelo 4: Gradient Boosting

Teoria: Ao contrário do Random Forest (paralelo/bagging), o Gradient Boosting treina árvores sequencialmente. Cada nova árvore tenta corrigir os erros da anterior. Tecnicamente, minimiza o gradiente de uma função de perda — daí o nome.
  • Geralmente mais preciso que Random Forest mas mais lento a treinar
  • Variantes populares: XGBoost, LightGBM, CatBoost (não incluídas mas compatíveis)
  • Hiperparâmetros chave: n_estimators, learning_rate, max_depth

8 Cross-Validation — Validação Honesta

Um simples split 80/20 pode ter sorte ou azar dependendo de quais dados caem no teste. Cross-Validation repete o processo N vezes com partições diferentes.

5-Fold Cross-Validation: Fold 1: [TEST] [TRAIN] [TRAIN] [TRAIN] [TRAIN] → R² = 0.87 Fold 2: [TRAIN] [TEST] [TRAIN] [TRAIN] [TRAIN] → R² = 0.84 Fold 3: [TRAIN] [TRAIN] [TEST] [TRAIN] [TRAIN] → R² = 0.88 Fold 4: [TRAIN] [TRAIN] [TRAIN] [TEST] [TRAIN] → R² = 0.85 Fold 5: [TRAIN] [TRAIN] [TRAIN] [TRAIN] [TEST] → R² = 0.86 Resultado: Mean R² = 0.860 ± 0.013 (muito mais confiável que um único split!) 
from sklearn.model_selection import cross_val_score

rf = RandomForestRegressor(n_estimators=50, random_state=42)
scores = cross_val_score(rf, X, y, cv=5, scoring='r2')

print(f"R² por fold: {scores}")       # [0.87, 0.84, 0.88, 0.85, 0.86]
print(f"Média: {scores.mean():.4f}")  # 0.8600
print(f"Std:   {scores.std():.4f}")   # 0.0130 — baixo = estável!

9 Aprendizagem Não-Supervisionada

Não temos labels/respostas certas. O algoritmo explora a estrutura interna dos dados para descobrir padrões escondidos.

Analogia: Imagina que recebes um ficheiro de 300 clientes sem qualquer etiqueta. Não sabes quem são "premium" ou "budget". Analisas o comportamento de compra e naturalmente notas grupos: uns compram muito e com frequência (champions), outros só em promoções (bargain hunters), outros raramente (lost customers). Isso é clustering!

Quando usar Não-Supervisionado?

  • Não tens labels históricas para treinar
  • Queres descobrir segmentos de mercado desconhecidos
  • Queres detectar anomalias/outliers
  • Queres reduzir dimensionalidade para visualização

10 Clustering — Agrupar Clientes

Features usadas por cliente: total_spent, avg_basket, num_transactions, avg_quantity, avg_discount.

Algoritmo 1: K-Means

Teoria: Divide N pontos em K grupos (clusters).
Funcionamento iterativo:
  1. Escolhe K centróides aleatórios
  2. Cada ponto é atribuído ao centróide mais próximo (distância Euclidiana)
  3. Actualiza centróides = média de todos os pontos no cluster
  4. Repete até convergir (centróides não mudam)
ITERAÇÃO 1: ITERAÇÃO FINAL (convergido): × × ★ × × × ★ × × ★ × → itera → × ★ × ★ ○ ○ ○ ★ ○ ○ ○ ○ ○ ○ ★ = centróide × = cluster A ○ = cluster B 
from sklearn.cluster import KMeans
from sklearn.metrics  import silhouette_score

# Determinar k óptimo com o "Elbow Method"
inertias = {}
for k in range(2, 9):
    km = KMeans(n_clusters=k, random_state=42, n_init=10)
    km.fit(X_scaled)
    inertias[k] = km.inertia_   # soma das distâncias ao centróide

# k onde a curva "dobra" = elbow = melhor k
# Neste projecto esperamos k=4 (os 4 segmentos originais)

kmeans = KMeans(n_clusters=4, random_state=42)
labels = kmeans.fit_predict(X_scaled)
sil = silhouette_score(X_scaled, labels)
print(f"Silhouette Score: {sil:.4f}")  # 0.0-1.0, maior = melhor

Elbow Method — Escolher o K óptimo

Inércia (soma das distâncias²) │ │\ │ \ │ \ ← "cotovelo" aqui = k óptimo (k=4 neste caso) │ \____ │ \____ └──────────────── k 2 3 4 5 6 7 8

Algoritmo 2: DBSCAN

Teoria (Density-Based Spatial Clustering of Applications with Noise): Em vez de centróides, baseia-se em densidade. Um cluster é uma região densa de pontos. Pontos isolados em regiões pouco densas são marcados como noise (-1) = outliers.
ParâmetroSignificadoNosso valor
epsRaio da vizinhança0.8 (em espaço normalizado)
min_samplesMínimo de pontos para formar cluster5

K-Means

  • Precisa de especificar K
  • Clusters esféricos
  • Todos os pontos têm cluster
  • Rápido, escalável

DBSCAN

  • K é automático
  • Clusters de qualquer forma
  • Detecta outliers (-1)
  • Sensível a eps e min_samples

11 PCA — Redução de Dimensionalidade

Temos 5 features de cliente. É impossível visualizar 5 dimensões num gráfico. O PCA (Principal Component Analysis) encontra as 2 direcções de maior variância e projeta todos os pontos nesse espaço 2D.

Matemática (simplificada): PCA calcula os autovectores da matriz de covariância dos dados. O primeiro autovector (PC1) aponta na direcção de maior variância. O segundo (PC2) é perpendicular ao PC1 e tem a segunda maior variância. Projectar em PC1+PC2 é como "achatar" os dados para o chão preservando o máximo de informação.
Dados originais em 5D → PCA → Projecção em 2D Antes: [total_spent, avg_basket, num_tx, avg_qty, avg_discount] ↓ PCA Depois: [PC1, PC2] PC1 = 0.72 × total_spent + 0.51 × avg_basket + 0.31 × num_tx + ... PC2 = -0.45 × total_spent + 0.62 × avg_basket + 0.41 × num_tx + ... Scatter 2D (muito mais interpretável!): PC2 │ Cluster 0 ●●● │ ●● Cluster 1 ○○○ │ ●●○○○ Cluster 2 ▲▲▲ │ ▲▲ Cluster 3 ■■■ └──────────── PC1 
from sklearn.decomposition import PCA

# Reduzir para 2 componentes principais
pca = PCA(n_components=2, random_state=42)
coords_2d = pca.fit_transform(X_scaled)   # shape: (n_customers, 2)

# Variância explicada
ev = pca.explained_variance_ratio_
print(f"PC1 explica {ev[0]*100:.1f}% da variância")
print(f"PC2 explica {ev[1]*100:.1f}% da variância")
print(f"Total: {sum(ev)*100:.1f}%")   # esperamos ~70-85%

# Ver todos os componentes (curva de scree)
pca_full = PCA()
pca_full.fit(X_scaled)
cumvar = np.cumsum(pca_full.explained_variance_ratio_)
# [0.45, 0.72, 0.88, 0.97, 1.00] → 3 componentes explicam 88%

12 Métricas de Avaliação — Tim-tim por Tim-tim

Métricas de Regressão

MAE
Mean Absolute Error
Erro médio absoluto em €.
MAE=2.5 → erro médio de €2.50 por linha
RMSE
Root Mean Squared Error
Penaliza erros grandes. Sempre ≥ MAE. Sensível a outliers.
R²
Coefficient of Determination
% de variância explicada. R²=0.85 → modelo explica 85% da variação
Fórmulas:
MAE = mean(|y - ŷ|)
RMSE = sqrt(mean((y - ŷ)²))
R² = 1 - SS_res / SS_tot onde SS_res = Σ(y-ŷ)², SS_tot = Σ(y-ȳ)²

Interpretação de R²

R² = 0.90+Excelente
R² = 0.75-0.90Bom
R² = 0.50-0.75Aceitável
R² < 0.50Fraco

Métricas de Clustering

Silhouette Score — Mede quão bem cada ponto está no seu cluster.
s = (b - a) / max(a, b)
onde: a = distância média ao cluster próprio; b = distância média ao cluster mais próximo.
Range: [-1, 1] — quanto mais alto melhor. Valores negativos indicam ponto no cluster errado.
Valor SilhouetteInterpretação
0.70 – 1.00Clusters muito bem definidos
0.50 – 0.70Clusters razoáveis
0.25 – 0.50Clusters fracos (pode haver sobreposição)
< 0.25Sem estrutura clara nos dados

14 OLAP vs OLTP — Diferenças e Quando Usar

🏪 OLTP — Online Transaction Processing

  • Schema: Normalizado (3NF)
  • Operações: INSERT, UPDATE, DELETE rápidos
  • Utilizadores: Milhares simultâneos
  • Queries: Simples, poucas linhas
  • Exemplo: POS de caixa, e-commerce checkout
  • Neste projecto: SQLite com transactions, sales_items

📊 OLAP — Online Analytical Processing

  • Schema: Desnormalizado (Star/Snowflake)

ETL — O elo entre OLTP e OLAP

ETL = Extract, Transform, Load
É o processo que move dados do OLTP (SQLite) para o OLAP (DuckDB). No nosso projecto, o setup_duckdb.py faz exactamente isto: lê do SQLite, transforma (join + cálculos), carrega no DuckDB. Em empresas grandes, isto corre em batch nocturnamente.

Funções Analíticas OLAP (Window Functions)

-- Running total de receita por mês (YTD)
SELECT
    month_date,
    revenue,
    SUM(revenue) OVER (
        ORDER BY month_date
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS ytd_revenue,
    revenue - LAG(revenue) OVER (ORDER BY month_date) AS mom_delta,
    RANK() OVER (ORDER BY revenue DESC) AS revenue_rank
FROM monthly_revenue;
Window FunctionO que faz
LAG(col, n)Valor da linha N posições atrás
LEAD(col, n)Valor da linha N posições à frente
SUM() OVER (...)Soma acumulada ou por janela
RANK() OVER (...)Ranking com saltos em empates
DENSE_RANK() OVER (...)Ranking sem saltos em empates
NTILE(n) OVER (...)Divide em N buckets iguais (usado no RFM)
ROW_NUMBER() OVER (...)Número sequencial único por janela

O que é PL/SQL?

PL/SQL = Procedural Language / SQL
Extensão procedural do SQL da Oracle. Permite variáveis, loops, condições, excepções, cursores — tudo dentro do servidor de base de dados. Ideal para:
  • Stored Procedures — lógica de negócio no servidor
  • Triggers — código que corre automaticamente em INSERT/UPDATE/DELETE
  • Packages — agrupar procedures + functions + tipos numa unidade
  • Cursors — iterar sobre resultados de queries linha a linha

Anatomia de um Package PL/SQL

-- SPEC = interface pública (o que é visível externamente)
CREATE OR REPLACE PACKAGE pkg_retail_analytics AS
    c_rfm_threshold CONSTANT INTEGER := 13;          -- constante

    FUNCTION  get_customer_label(p_id IN INTEGER) RETURN VARCHAR2;
END pkg_retail_analytics;

RFM Analysis com PL/SQL + OLAP

RFM = Recency, Frequency, Monetary
Método clássico para segmentar clientes com base em 3 dimensões:
  • Recency: Há quantos dias foi a última compra? (menos = melhor)
  • Frequency: Quantas transacções fez? (mais = melhor)
  • Monetary: Quanto gastou no total? (mais = melhor)
Cada dimensão é pontuada de 1-5 com NTILE(5). Score total 13-15 = Champion.

Materialised Views — Performance OLAP

Problema: Uma query que soma 10M de linhas demora 30 segundos.
Solução: Materialised View pré-computa o resultado e guarda fisicamente. A query passa a demorar <1 segundo. Actualiza-se em batch (nightly refresh).

Partitioning — Escalabilidade OLAP

-- Sem particionamento: scanneia 10M linhas para encontrar Q4
SELECT * FROM fct_sales_olap WHERE date_id BETWEEN 20241001 AND 20241231;

-- COM particionamento: Oracle salta directamente para p_2024_q4
-- Partition pruning: 75% das linhas ignoradas automaticamente!
CREATE TABLE fct_sales_partitioned ...
PARTITION BY RANGE (date_id) (
    PARTITION p_2024_q4 VALUES LESS THAN (20250101),
    ...
