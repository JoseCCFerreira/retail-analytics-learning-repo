Retail Analytics Case

Documento unico para criar, entender, executar e publicar um projeto de dados com OLTP, OLAP, dbt, Streamlit, Machine Learning, SQL, PL/SQL, Git e GitHub.

Problema Estrutura Instalacao OLTP OLAP dbt Streamlit ML Deep Learning PL/SQL GitHub Atalhos

1. O Problema

Uma empresa de retalho quer perceber melhor as suas vendas. Para isso precisa de guardar transacoes, transformar dados, criar metricas, visualizar resultados e experimentar modelos de Machine Learning.

Este projeto simula esse caminho completo. Comeca com dados operacionais, cria uma base analitica, governa transformacoes com dbt, mostra dashboards em Streamlit e treina modelos simples para prever valores e segmentar clientes.

OLTP Base do dia a dia. Guarda clientes, produtos, lojas, transacoes e itens vendidos.
OLAP Base para analise. Junta dados e prepara tabelas para metricas e dashboards.
dbt Governa transformacoes SQL, testes e modelos finais.
Streamlit Cria visualizacoes interativas em Python.
Machine Learning Usa scikit-learn para prever vendas, classificar clientes e descobrir grupos.

4. OLTP Database - SQLite

OLTP significa Online Transaction Processing. E o tipo de base de dados para operacoes normais do negocio: inserir vendas, consultar clientes, guardar produtos e registar compras.

Passo 1 - Gerar CSVs

python python/generate_data.py

Este script cria ficheiros como:

  • data/customers.csv
  • data/products.csv
  • data/stores.csv
  • data/transactions.csv
  • data/sales_items.csv

Passo 2 - Criar SQLite

python python/setup_sqlite.py

Codigo principal do esquema OLTP

CREATE TABLE customers (
    customer_id INTEGER PRIMARY KEY,
    customer_name TEXT NOT NULL,
    segment TEXT NOT NULL,
    city TEXT NOT NULL
);

CREATE TABLE transactions (
    transaction_id INTEGER PRIMARY KEY,
    transaction_date TEXT NOT NULL,
    customer_id INTEGER NOT NULL,
    store_id INTEGER NOT NULL,
    payment_method TEXT NOT NULL,
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id),
    FOREIGN KEY (store_id) REFERENCES stores(store_id)
);

A chave primaria identifica cada linha. A chave estrangeira liga tabelas. Por exemplo, transactions.customer_id liga uma venda a um cliente.

5. OLAP Database - DuckDB

OLAP significa Online Analytical Processing. Serve para analise, agregacoes, tendencias, dashboards e modelos. Aqui usamos DuckDB porque e simples, local e muito bom para analytics.

Criar a base analitica

python python/setup_duckdb.py

Tabela de factos

A tabela fct_sales e a tabela central. Ela junta a transacao com as linhas de venda. Cada linha representa um produto vendido dentro de uma transacao. 

CREATE TABLE fct_sales AS
SELECT
    si.sales_item_id,
    t.transaction_id,
    CAST(t.transaction_date AS DATE) AS transaction_date,
    t.customer_id,
    t.store_id,
    t.payment_method,
    si.product_id,
    si.quantity,
    si.unit_price,
    si.discount_pct,
    si.gross_amount,
    si.discount_amount,
    si.net_amount
FROM sales_items si
INNER JOIN transactions t
    ON t.transaction_id = si.transaction_id;

Exemplo de pergunta OLAP

SELECT
    date_trunc('month', transaction_date) AS month_date,
    ROUND(SUM(net_amount), 2) AS revenue
FROM fct_sales
GROUP BY 1
ORDER BY 1;

6. dbt - Governar os Dados

dbt permite escrever transformacoes em SQL, versionar essas transformacoes no Git e testar a qualidade dos dados. Neste projeto, dbt trabalha sobre o DuckDB.

Camadas usadas

Camada Objetivo
staging Preparar dados de origem, normalizar nomes e tipos.
marts Criar tabelas finais para BI, dashboards e ML.
schema.yml Definir testes como not null, unique e relacionamentos.

Executar dbt

python python/run_pipeline.py
cd dbt_retail_analytics
cp profiles.example.yml profiles.yml
export DBT_PROFILES_DIR=.
dbt run
dbt test

Exemplo de modelo dbt

select
    si.sales_item_id,
    t.transaction_id,
    t.transaction_date,
    t.customer_id,
    t.store_id,
    t.payment_method,
    si.product_id,
    si.quantity,
    si.net_amount
from {{ ref('stg_sales_items') }} si
inner join {{ ref('stg_transactions') }} t
    on t.transaction_id = si.transaction_id

Exemplo de teste dbt

models:
  - name: fct_sales
    columns:
      - name: sales_item_id
        tests: [not_null, unique]
      - name: transaction_id
        tests:
          - not_null
          - relationships:
              arguments:
                to: ref('stg_transactions')
                field: transaction_id
Porque isto e governanca? Porque deixa claro que tabelas existem, que regras devem cumprir e que transformacoes criam as tabelas finais.

8. Machine Learning

O objetivo de ML aqui nao e criar um sistema perfeito. O objetivo e aprender quais tipos de modelos existem e o que cada um tenta responder num problema de retalho.

Executar ML

python python/run_pipeline.py
python python/ml_retail.py

Resultados gerados

  • data/ml_outputs/model_metrics.json: metricas dos modelos.
  • data/ml_outputs/regression_feature_importance.csv: variaveis mais importantes.
  • data/ml_outputs/cluster_assignments.csv: grupos de clientes.
  • data/ml_outputs/pca_projections.csv: pontos para grafico 2D.

Modelos explicados

Modelo Tipo O que pretendo com ele
Linear Regression Regressao Prever o valor liquido de uma linha de venda de forma simples e comparavel.
Decision Tree Regressor Regressao Perceber regras do tipo: se preco alto e desconto baixo, entao valor esperado maior.
Random Forest Regressor Regressao Melhorar previsao usando varias arvores e obter importancia das variaveis.
Gradient Boosting Regressor Regressao Testar um modelo forte que aprende com erros sucessivos.
Logistic Regression Classificacao Prever o segmento do cliente com uma baseline simples.
Decision Tree Classifier Classificacao Classificar clientes com regras mais interpretaveis.
Random Forest Classifier Classificacao Classificar segmentos com mais estabilidade.
K-Means Clustering Descobrir grupos de clientes parecidos sem usar etiquetas pre-definidas.
DBSCAN Clustering Encontrar grupos por densidade e marcar outliers.
PCA Reducao dimensional Transformar varias variaveis em duas componentes para visualizar clientes num grafico.

Metricas simples

  • MAE: erro medio em euros. Quanto menor, melhor.
  • RMSE: erro que penaliza erros grandes. Quanto menor, melhor.
  • R2: explica a variacao dos dados. Mais perto de 1 e melhor.
  • Accuracy: percentagem de classificacoes corretas.
  • Silhouette Score: qualidade dos clusters. Mais alto e melhor.

9. Deep Learning com PyTorch e TensorFlow

Depois de dominares scikit-learn, podes evoluir para Deep Learning. Isso ajuda-te a aprender redes neuronais, embeddings, previsão temporal mais rica e problemas de recomendação.

Ferramentas recomendadas

  • PyTorch: melhor para aprender a mecânica do treino.
  • TensorFlow / Keras: ótimo para estruturar treino de forma mais guiada.
  • JupyterLab: ideal para estudo e experiências.
  • MLflow: útil para guardar runs e métricas.

Instalação opcional

pip install -r requirements_deep_learning.txt

Metodologia correta

  1. Começar com baseline simples.
  2. Definir métrica antes do treino.
  3. Testar rede neuronal pequena.
  4. Comparar honestamente com o baseline.
  5. Só manter Deep Learning se realmente trouxer valor.

10. Ajuda com PL/SQL

PL/SQL e a linguagem procedural da Oracle. SQLite e DuckDB usam SQL, mas nao executam PL/SQL. Neste projeto a pasta sql/oracle_plsql_concepts serve para estudo de conceitos Oracle, especialmente triggers, procedures, exceptions e diferencas OLTP/OLAP.

Bloco PL/SQL minimo

SET SERVEROUTPUT ON;

DECLARE
    v_total NUMBER;
BEGIN
    SELECT COUNT(*)
    INTO v_total
    FROM transactions;

    DBMS_OUTPUT.PUT_LINE('Total de transacoes: ' || v_total);
EXCEPTION
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('Erro: ' || SQLERRM);
END;
/

Procedure exemplo

CREATE OR REPLACE PROCEDURE show_customer_count AS
    v_total NUMBER;
BEGIN
    SELECT COUNT(*)
    INTO v_total
    FROM customers;

    DBMS_OUTPUT.PUT_LINE('Clientes: ' || v_total);
END;
/

EXEC show_customer_count;

Trigger exemplo

CREATE OR REPLACE TRIGGER trg_check_quantity
BEFORE INSERT OR UPDATE ON sales_items
FOR EACH ROW
BEGIN
    IF :NEW.quantity <= 0 THEN
        RAISE_APPLICATION_ERROR(-20001, 'Quantity must be greater than zero');
    END IF;
END;
/

Como executar PL/SQL

  1. Instalar Oracle Database ou usar um ambiente Oracle ja existente.
  2. Abrir SQL Developer ou SQLcl.
  3. Ativar output com SET SERVEROUTPUT ON;.
  4. Executar scripts terminados com / quando forem blocos, procedures ou triggers.
