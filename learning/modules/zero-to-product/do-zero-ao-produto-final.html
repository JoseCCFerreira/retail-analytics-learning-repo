Phase 0: Preparar ambiente ⏱ 8 min

✓

Verificar ferramentas básicas no terminal

Setup

Começa por confirmar versões instaladas. Sem este check inicial, os passos seguintes podem falhar sem mensagem clara.

Do

Terminal 
python --version
git --version
node --version
npm --version

Verify

Verificação comportamental. O terminal deve devolver uma linha por comando com número de versão. Este retorno confirma que as ferramentas estão acessíveis no PATH.

Instala Python 3.10+ e ativa a opção de adicionar Python ao PATH no instalador. Depois fecha e abre o terminal.

✓

Criar ambiente virtual e instalar dependências

Setup

Agora prepara o ambiente Python local. Este ambiente guarda todas as bibliotecas usadas pelo projeto.

Do

Windows PowerShell 
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt
macOS / Linux 
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt

Verify

Confirmar. O comando pip install -r requirements.txt termina sem erro, e o prompt mostra o prefixo da virtualenv ativa.
🛠

Executar modelagem dbt

Generate

A modelagem dbt transforma dados em tabelas de análise. Se não correres este passo, os modelos marts podem não existir.

Do

Terminal 
cd dbt_retail_analytics
cp profiles.example.yml profiles.yml
set DBT_PROFILES_DIR=%cd%   # Windows cmd
# $env:DBT_PROFILES_DIR = (Get-Location).Path   # PowerShell
# export DBT_PROFILES_DIR=$(pwd)               # macOS/Linux
dbt deps
dbt seed
dbt run
dbt test
cd ..

Verify

Confirmar. O terminal mostra que run e test terminaram com sucesso e sem falhas críticas.

Em PowerShell usa $env:DBT_PROFILES_DIR = (Get-Location).Path. Em bash usa export DBT_PROFILES_DIR=$(pwd).
